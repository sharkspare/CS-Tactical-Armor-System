# STM32G474 硬件资源利用手册

## 一、文档用途

这份手册用于说明当前 `NUCLEO-G474RE` 工程中各个硬件资源的实际占用情况，重点回答三个问题：第一，当前每个模块接到了哪些 MCU 引脚；第二，外部模块应该怎样与开发板连接；第三，哪些引脚已经被占用，后续扩展时尽量不要随意改动。

本文档以当前工程 `myproject_G474_base` 的 `.ioc` 配置和代码定义为准，适合作为后续接线、排错、增减模块和整理作品说明时的硬件参考。

## 二、当前平台

当前主控板卡为 `NUCLEO-G474RE`，主控芯片为 `STM32G474RE`。工程当前主频为 `170 MHz`，并已经启用了板载虚拟串口、板载按键和板载 LED 资源。

当前工程已经实际接入或预留的外设包括调试串口、GPS、语音串口、MPU 惯性模块、红外接收、振动马达、ST7735 LCD 和 `NRF24L01` 无线模块。

## 四、按模块看接线关系

这一部分按“外部模块怎么接”来写，实际接线时可以直接照着对应模块去找。

### 1. 调试串口与电脑连接

调试串口不需要你额外接杜邦线。`NUCLEO-G474RE` 自带 `ST-LINK` 虚拟串口链路，当前工程把日志输出放在板载虚拟串口上，也就是 `PA2/PA3` 这对口。你只需要用开发板的 USB 连上电脑，在串口助手中打开对应虚拟串口，就可以看到 `printf` 输出。

这一组引脚虽然在芯片内部是 `PA2/PA3`，但在日常使用中应当视为“板载调试专用资源”。如果后续还需要稳定调试，尽量不要把它们改去接外部模块。

### 2. 板载 LED 与板载按键

板载绿色 LED 使用 `PA5`，这是当前工程中最基础的运行指示资源。程序运行后它会按固定节拍闪烁，用来表示主循环仍在工作。

板载按键 `B1` 使用 `PC13`，当前工程中它承担两个用途：一是按键本身测试；二是切换 LCD 页面。因为它是板载现成资源，所以不需要外接按键模块。后续如果还想保留“按一下就切页”的交互方式，`PC13` 最好不要再复用到别的地方。

### 3. GPS 模块接线

GPS 当前使用 `USART3`，波特率为 `9600 8N1`。接线关系如下：

| GPS 模块引脚 | 连接到 G474 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| `TXD` | `PB11` | 必接 | GPS 发数据，STM32 接收 |
| `RXD` | `PB10` | 可选 | 如果将来需要给 GPS 发配置命令再接 |
| `GND` | `GND` | 必接 | 共地 |
| `VCC` | 按模块规格供电 | 必接 | 优先确认模块板上标注 |

这里最关键的一点是：如果你只是要接收 GPS 定位信息，那么 `TXD -> PB11` 是最核心的一根线。`RXD -> PB10` 可以先不接，但建议在正式整理接线时也一并预留。

需要特别注意电平问题。GPS 模块如果输出的是 `3.3V TTL` 串口，可以直接接入；如果模块串口是 `5V TTL`，建议加电平转换后再接入 MCU。

### 4. 语音模块串口接线

语音模块当前使用 `USART1`，波特率同样为 `9600 8N1`。接线关系如下：

| 语音模块引脚 | 连接到 G474 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| `TX` | `PA10` | 必接 | 语音模块发送，STM32 接收 |
| `RX` | `PA9` | 必接 | STM32 发送，语音模块接收 |
| `GND` | `GND` | 必接 | 共地 |
| `VCC` | 按模块规格供电 | 必接 | 供电电压以模块实物标识为准 |

这一路串口是当前系统与语音模块交互的专用口，不建议和 GPS 共用，也不建议临时改去做调试口。调试输出已经有板载虚拟串口，语音模块保持独立会更稳。

### 5. MPU6050/MPU6500 类惯性模块接线

当前工程中的惯性模块采用软件 `I2C`，引脚固定为 `PB6/PB7`。接线关系如下：

| MPU 模块引脚 | 连接到 G474 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| `SCL` | `PB6` | 必接 | 软件 I2C 时钟 |
| `SDA` | `PB7` | 必接 | 软件 I2C 数据 |
| `VCC` | 按模块规格供电 | 必接 | 优先使用与逻辑兼容的供电方式 |
| `GND` | `GND` | 必接 | 共地 |
| `INT` | 当前未使用 | 可不接 | 如后续做中断采样可再接 |
| `AD0` | 当前未强制使用 | 可先不接 | 是否接地/拉高取决于模块实际地址设计 |
| `XDA` | 当前未使用 | 不接 | 保持悬空即可 |
| `XCL` | 当前未使用 | 不接 | 保持悬空即可 |

当前工程代码按 `0x68` 地址访问这类器件，但实测模块 `WHO_AM_I` 返回过 `0x70`，说明你手上的器件更接近 `MPU6500-class` 模块。这个现象是正常的，只要初始化和读数稳定，就不影响当前工程继续使用。

在接线实践中，`SCL` 和 `SDA` 必须接对，接反时最容易出现“初始化失败、读 ID 错误、全是 `0xFF`”这种现象。当前工程对 `PB6/PB7` 已经配置为开漏加上拉，所以尽量不要再把这两个脚改成推挽输出用途。

### 6. 红外接收模块接线

红外接收模块当前只用了一个信号脚，连接很简单：

| 红外模块引脚 | 连接到 G474 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| `OUT` / `DO` | `PB9` | 必接 | 当前工程按低电平有效处理 |
| `VCC` | 按模块规格供电 | 必接 | 常见模块可用 `3.3V` 或 `5V`，以模块板为准 |
| `GND` | `GND` | 必接 | 共地 |

当前工程对这个输入采用轮询判定，并在软件中实现了锁存和冷却时间，因此硬件上只需要保证输出稳定、接地可靠即可。如果模块空闲时输出高电平、触发时输出低电平，那么就和当前工程逻辑匹配。

### 7. 振动马达接线

当前工程把振动马达控制信号放在 `PA8`，逻辑是高电平有效。但这里一定要注意，`PA8` 更适合作为“控制信号”，不适合直接硬拉一个直流振动马达。

更合理的接法是：

| 控制链路 | 接法建议 |
| --- | --- |
| `PA8` | 接三极管基极电阻或 MOSFET 栅极 |
| 马达一端 | 接电源正极 |
| 马达另一端 | 接三极管/MOSFET 受控端 |
| 驱动器地 | 接系统 `GND` |
| 马达电源地 | 与 STM32 共地 |

如果你用的是带驱动板的小型振动模块，那么 `PA8` 可以直接接它的控制输入脚；如果你用的是裸振动马达，建议必须加三极管或 MOSFET，并在电机两端加续流保护。简单说，`PA8` 是“命令脚”，不是“电机电源脚”。

### 8. ST7735 LCD 接线

你现在用的是 `1.8' 128x160 RGB TFT LCD`，驱动芯片为 `ST7735`。当前工程按 `SPI2` 模式连接，接线关系如下：

| LCD 模块引脚 | 连接到 G474 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| `BL` | 常亮接电源，当前未进 MCU | 一般接 | 当前工程没有单独背光控制脚 |
| `CS` | `PB12` | 必接 | 片选 |
| `DC` | `PC6` | 必接 | 数据/命令选择 |
| `RES` | `PC7` | 必接 | 硬件复位 |
| `SDA` | `PB15` | 必接 | 这里是 SPI `MOSI`，不是 I2C 的 `SDA` |
| `SCL` | `PB13` | 必接 | 这里是 SPI 时钟 `SCK` |
| `VCC` | 建议 `3.3V` | 必接 | 最稳妥做法是 3.3V 供电 |
| `GND` | `GND` | 必接 | 共地 |

有几点很容易接错。第一，很多这种小屏模块上写着 `SDA`、`SCL`，但它实际工作在 `SPI` 模式，此时 `SDA` 不是 I2C 数据线，而是 `SPI MOSI`。第二，当前工程没有使用 LCD 的回读功能，所以 `PB14` 虽然在 `CubeMX` 中被配置成了 `SPI2_MISO`，但模块侧通常可以不接 `MISO`。第三，`BL` 当前没有接到 MCU，因此背光一般是上电常亮模式。

如果你后续希望让屏幕背光可控，可以再单独找一个空闲 GPIO 加一条 `BL` 控制线，但当前工程还没有这样做。

### 9. NRF24L01 无线模块接线

`NRF24L01` 当前使用 `SPI3` 独立连接，接线关系如下：

| NRF24L01 引脚 | 连接到 G474 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| `VCC` | `3.3V` | 必接 | 绝对不要直接接 `5V` |
| `GND` | `GND` | 必接 | 共地 |
| `CE` | `PB0` | 必接 | 收发使能 |
| `CSN` | `PA4` | 必接 | SPI 片选 |
| `SCK` | `PC10` | 必接 | SPI 时钟 |
| `MOSI` | `PC12` | 必接 | 主发从收 |
| `MISO` | `PC11` | 必接 | 主收从发 |
| `IRQ` | `PB1` | 建议接 | 当前工程保留并读取该脚状态 |

这部分有两条非常重要的硬件经验。第一，`NRF24L01` 一定要用 `3.3V` 供电，不要偷懒接 `5V`。第二，`NRF24L01` 对供电稳定性比较敏感，建议在模块电源附近加一个 `10uF` 左右的电容，很多“能识别但收发不稳定”的问题其实都和供电纹波有关。

## 五、板载资源与外接资源的边界

为了后续开发更清楚，这里把“板载现成资源”和“外部接线资源”分开说明。

板载现成资源包括：

| 资源 | 对应引脚 | 当前作用 |
| --- | --- | --- |
| 板载虚拟串口 | `PA2/PA3` | 调试日志 |
| 板载 LED `LD2` | `PA5` | 心跳指示 |
| 板载按键 `B1` | `PC13` | 切页与按键测试 |
| SWD 下载调试 | `PA13/PA14` | 下载与在线调试 |
| SWO 调试 | `PB3` | 当前保留 |

外部模块接线资源包括：

| 模块 | 主要占用引脚 |
| --- | --- |
| GPS | `PB10/PB11` |
| 语音模块 | `PA9/PA10` |
| MPU 惯性模块 | `PB6/PB7` |
| 红外接收 | `PB9` |
| 振动马达控制 | `PA8` |
| ST7735 LCD | `PB12/PB13/PB15/PC6/PC7` |
| NRF24L01 | `PA4/PB0/PB1/PC10/PC11/PC12` |

这样区分之后，后续如果你要扩展功能，就能更快判断某个脚到底是“板子自带就占了”，还是“当前工程业务逻辑占了”。

## 六、哪些引脚尽量不要改

下面这些引脚建议在当前阶段尽量不要乱动，否则很容易破坏现有系统：

`PA2/PA3`
这组是当前唯一稳定好用的板载调试串口。改掉之后，很多日志都看不到。

`PA13/PA14`
这是下载和在线调试接口。除非你非常明确要改成别的下载方式，否则不要动。

`PC13`
这是板载按键。当前 LCD 切页已经依赖它，改掉之后界面交互会受影响。

`PB6/PB7`
这是 MPU 的软件 I2C。当前惯性姿态与动作识别都依赖它。

`PB12/PB13/PB15/PC6/PC7`
这是 LCD 整套控制链路。动其中任何一根线，屏幕都可能直接黑屏或花屏。

`PC10/PC11/PC12/PA4/PB0/PB1`
这是 `NRF24L01` 整套资源。当前无线功能已经建立在这组线上。

## 七、供电与接线注意事项

第一，所有外部模块必须与开发板共地。很多“串口没数据”“屏幕偶尔花”“无线不稳定”“I2C 识别不到”的问题，本质上都是共地没做好。

第二，能用 `3.3V` 逻辑就尽量统一到 `3.3V`。尤其是 `NRF24L01` 和 MCU 直接相连的通信线，最好全部保持 `3.3V` 体系。

第三，振动马达不要默认直接挂在 MCU IO 口上。即使程序里只是输出高低电平，硬件层也最好经过驱动器件。

第四，LCD 模块上写的 `SDA/SCL` 不一定是 `I2C`。在你现在这块 `ST7735` 屏上，它们实际上对应的是 `SPI MOSI/SCK`。

第五，`PB14` 目前虽然挂在 `SPI2_MISO` 上，但你的 LCD 没有实际用到它。如果将来真的非常缺引脚，可以考虑在 `CubeMX` 里把 `SPI2` 改成更适合只发不收的模式，再回收 `PB14`。但这属于后续优化，不建议在当前稳定版本上随便动。

## 八、后续扩展时的建议

如果后面你还要继续接新模块，建议优先遵守下面这个思路。能继续走独立串口的，就尽量不要和 GPS、语音共口；能继续走空闲普通 GPIO 的，就不要去碰已经稳定工作的 LCD、MPU、NRF 资源；如果要接功率器件，例如蜂鸣器、继电器、马达、灯带，默认都先考虑“GPIO 只出控制信号，外部加驱动”。

当前这套硬件资源分布最大的优点，是各个模块边界已经比较清楚。只要后续扩展时不去硬挤现有总线，不去破坏板载调试链路，这个工程还可以继续稳定地往上加功能。
